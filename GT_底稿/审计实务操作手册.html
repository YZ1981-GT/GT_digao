底稿编号解读

B 23 - 2 - 3 - 4 │ │ │ │ │ │ │ │ │ └─ 四级：子底稿序号 │ │ │ └─────── 三级：二级子底稿序号 │ │ └────────────── 二级：循环序号 (2=货币资金) │ └────────────────────── 一级：底稿系列号 (23=业务层面控制) └─────────────────────────── 类别代码 (B=业务控制, C=控制测试, D-M=实质性)
📝
快速记忆口诀：B = Business（业务控制）| C = Control（控制测试）| D-M = Detail（细节测试，按循环字母排序）

是否需要执行控制测试？

被审计单位有内部控制吗？ ├─ 否 → 直接实质性测试（不用C类底稿） └─ 是 → 穿行测试发现控制设计有效吗？ ├─ 否 → 直接实质性测试 └─ 是 → 你打算依赖控制以降低实质性测试吗？ ├─ 否 → 只做有限控制测试或直接实质性测试 └─ 是 → 执行控制测试（使用C类底稿）
✅
需要做控制测试：大型企业、内控完善、控制可测试、打算依赖控制
❌
不需要控制测试：小型企业、内控薄弱、控制不可测试、初次审计风险高

🧪 控制测试阶段

测试关键控制运行有效性，确定可依赖的控制，调整实质性测试范围

控制测试样本量速查表

根据控制频率和风险等级确定样本量
控制频率低风险（偏差率0-1%）中风险（偏差率1-3%）高风险（偏差率3-5%）
每笔交易25个30-35个40个
每天25个30-35个40个
每周15个20-25个25个
每半月10个15个20个
每月5个8-10个全部
每季度2-3个全部全部
每年1个全部全部
⚠️
样本量调整因素：初次审计 +20~40% | 复杂控制 +10~30% | 前期发现问题 +20~40% | 自动化控制 -10%

偏差率判断标准

偏差率控制有效性评价对审计策略的影响
0-2%控制有效可依赖控制，降低实质性测试范围
3-5%控制基本有效部分依赖，实质性测试正常范围
6-10%控制部分失效不依赖控制，增加实质性测试
>10%控制失效控制不可依赖，充分实质性测试

测试方法选择

检查（Inspection）

适用于有书面证据的控制。获取控制执行的单据/记录，核对控制是否按要求执行，关注审批签名、日期、金额等关键要素。

询问（Inquiry）

适用于了解控制执行情况。向控制执行人员询问，关注控制的一致性和例外情况，必要时进行交叉验证。

观察（Observation）

适用于现场执行的控制。现场观察控制执行过程，关注控制是否按流程执行，拍照或录像留存证据。

重新执行（Reperformance）

适用于验证控制有效性。独立重新执行控制程序，将结果与被审计单位结果对比，评估控制可靠性。

📊 实质性测试阶段

发现认定层次重大错报，获取充分适当审计证据，形成审计结论

实质性测试范围调整

控制测试结论范围调整具体措施
控制有效（≤2%）降低20%-30%减少样本量，降低详细程度
基本有效（3-5%）保持正常范围按计划执行
部分失效（6-10%）增加20%-30%增加样本量，提高详细程度
控制失效（>10%）增加50%以上大幅增加测试范围，加强检查

实质性测试程序类型

细节测试

  • 余额核对：获取明细账与总账核对
  • 交易测试：抽取样本检查支持性单据
  • 截止测试：检查截止期前后交易
  • 计价测试：获取计价依据重新计算
  • 分类测试：检查科目分类和列报

分析程序

  • 趋势分析：同比、环比分析
  • 比率分析：财务比率计算对比
  • 合理性测试：估算并与账面对比
  • 配比分析：关联科目配比关系

函证程序

  • 银行函证：存款、借款、票据、理财
  • 客户函证：应收账款余额
  • 供应商函证：应付账款余额
  • 律师函证：诉讼、担保事项

监盘程序

  • 现金盘点：突击盘点、现场清点
  • 存货监盘：抽盘、观察、询问
  • 固定资产：抽查、观察、拍照
  • 银行存单：盘点、核对、函证

❓ 常见问题处理

审计实务中常见问题的快速解决方案

Q1：第一次执行审计项目，从哪里开始？▶
  1. 了解被审计单位 — 阅读项目立项资料、上期审计报告
  2. 识别重要业务循环 — 看资产负债表，识别重要科目，确定5-8个循环
  3. 执行穿行测试 — 针对每个业务循环，使用B类底稿
💡
新手建议：先从简单的循环入手，如货币资金、固定资产，积累经验后再做复杂的收入、存货循环。
Q2：穿行测试和控制测试有什么区别？▶
对比维度穿行测试控制测试
目的了解控制设计测试控制运行有效性
阶段风险评估阶段控制测试阶段
样本量1-2笔交易20-25笔（根据频率）
底稿B类底稿（B23-X-4）C类底稿（CX）
结论控制设计有效/无效控制运行有效/无效
Q3：函证未回函怎么办？▶
  1. 催函 — 第一次发函后15天未回，再次催函
  2. 二次催函 — 再等10天仍未回，第三次催函或电话催促
  3. 执行替代程序：
    • 应收账款：检查期后回款（建议100%回款）
    • 银行存款：获取银行对账单、流水
    • 应付账款：检查期后付款
  4. 记录底稿 — 说明未回函原因和替代程序
  5. 评估影响 — 如无法获取充分证据，考虑保留意见
⚠️
替代程序的证据力度低于函证，需增加测试样本量。如未回函比例过高（>30%），考虑对审计意见的影响。
Q4：发现错报如何处理？▶
错报金额 < 明显微小错报阈值（通常为PM的5-10%） → 可以不要求调整，记录未调整差异 错报金额 ≥ 明显微小阈值 且 < 实际执行重要性 → 建议调整，如管理层不调整则累积错报 错报金额 ≥ 实际执行重要性 → 强烈建议调整，可能影响审计意见
🔴
涉及舞弊的错报，无论金额大小，都应要求调整。影响关键指标（如扭亏为盈）的错报也建议调整。
Q5：监盘时发现账实不符怎么处理？▶
差异类型处理措施审计调整
盘盈查明原因（入账错误？遗漏？），要求管理层调整，评估内控缺陷建议调整
盘亏查明原因（盗窃？毁损？记录错误？），检查审批程序，评估损失处理重大差异必须调整
状态异常识别呆滞、毁损存货，评估减值准备充分性，拍照留证建议计提减值

📝 典型错误案例库

审计实务中的典型错误案例分析与审计发现方法

收入循环错误案例

案例1：收入确认时点错误▶

错误情况

企业按发货时点确认收入，但合同约定需客户验收后才转移控制权。期末大量已发货未验收的商品确认了收入。

会计影响

  • 虚增收入3000万元
  • 虚增应收账款3000万元
  • 虚减存货成本2200万元
  • 虚增利润800万元

审计发现方法

  • 检查销售合同，识别控制权转移时点
  • 执行收入截止测试，关注期末大额发货
  • 检查期后退货情况
  • 询问业务人员收入确认政策

正确处理

应在客户验收时确认收入，已发货未验收应计入"发出商品"，成本暂不结转。

案例2：虚构销售收入▶
🔴
舞弊特征：关联方交易、缺少物流单据、期后大额退货、毛利率异常（接近0%）

向关联方虚构销售500万元，没有实际货物交付，期后全额退货。审计发现方法：识别关联方清单、检查物流单据、执行期后退货检查、分析毛利率、实地走访客户。

存货循环错误案例

案例3：存货计价错误▶

存货采用先进先出法，系统设置错误按后进先出计算。原材料价格上涨期间，导致虚减营业成本500万元、虚增存货500万元。审计发现方法：检查存货计价政策、重新计算存货成本、分析毛利率变动、测试系统计价逻辑。

案例4：存货跌价准备计提不足▶

存货中包含大量呆滞品（3年以上未动），市场价格已跌破成本30%，企业未计提跌价准备。虚增存货1000万元、虚增利润1000万元。审计发现方法：监盘时识别呆滞存货、获取库龄表、检查市场价格进行跌价测试、检查期后销售情况。

固定资产循环错误案例

案例5：在建工程长期未转固▶

生产线2年前已达到预定可使用状态，企业一直挂在建工程未转固定资产，未计提折旧。应转固金额5000万元，少计折旧500万元/年，2年累计虚增利润1000万元。审计发现方法：检查在建工程明细账、询问工程部门项目进度、实地观察状态、检查工程验收报告。

案例6：固定资产折旧年限不合理▶

服务器设备折旧年限设为10年（行业一般3-5年），每年少计折旧95万元。审计发现方法：检查折旧政策对比行业惯例、询问资产管理部门实际使用寿命、检查同类资产报废更新情况。

负债与权益循环错误案例

案例7：借款费用资本化错误▶

企业将所有利息费用均资本化至在建工程，包括流动资金借款利息。不应资本化的利息300万元，虚增在建工程和利润。审计发现方法：获取借款费用资本化计算表、检查借款合同识别专门借款和一般借款、验证资本化期间、重新计算资本化率。

案例8：盈余公积计提不足▶

企业净利润1000万元，应计提法定盈余公积100万元（10%），实际只计提了50万元。审计发现方法：获取利润分配方案和股东会决议、重新计算应提取的盈余公积、核对公司章程规定的计提比例。

  • 关联方交易的公允性
  • 关联方交易的披露完整性

4检查披露完整性（附注核对）

关联方识别方法

识别方法信息来源关注要点

管理层声明管理层提供的关联方清单完整性确认

银行流水银行对账单异常大额往来对手方
🔴
舞弊高发：通过关联方虚构收入、关联方资金占用、隐瞒关联方关系、非公允关联交易。IPO项目必须100%检查所有关联方交易。
